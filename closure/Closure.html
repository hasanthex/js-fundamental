What is closures In general Colsure is a function which can access from outside the scope.
Advantages
  • Provide data encapsulation. It was possible to emulate private methods using closures.
  • Remove code redandancy.
  • That allow to attach variables to an execution context.
Disadvantages
  • Too many closures can slow down the application.
  • The variables declared inside a closure are not garbage collected.
Without Closure
let sum = 0
function add(num){
  sum = sum + num
}
add(1)
add(2)
console.log(sum)
Output
3

At this point all are looks good. our code running well and give us our expected output.
let sum = 0
function add(num){
  sum = sum + num
}
add(1)
add(2)
console.log(sum)

add(3)
sum = -1
console.log(sum)
Output
3
-1

Here is a big problem. we can change value of sum anywhere of this scope.
To overcome this kind of problem we need to use JS closures.

With Closure
function series(){
  let sum = 0
  function add(num){
    return sum = sum + num
  }
  return add
}

let value = series()
value(1)
value(2)
console.log(value(3))
Output
6

Here all ok now. Now our variable can't effected form the outter scope.
